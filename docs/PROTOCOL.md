# LORDDOM LGUB 초음파 거리센서 — Modbus-RTU 프로토콜 (역설계 명세)

> 제조사가 레지스터맵/프로토콜 문서를 공개하지 않아, 실물 프로빙으로 역설계해
> 작성한 비공식 명세다. 실측·데이터시트 교차검증 결과를 근거로 기록한다.
>
> - 대상 개체: **LGU1000-18GM55-R4-V15** (측정 60~1000mm, RS485)
> - 검증 환경: Ubuntu, FTDI FT232 USB-RS485 어댑터, `/dev/ttyUSB0`
> - 검증일: 2026-07-09
> - 신뢰도 기준: 배율은 데이터시트 분해능(0.17mm) 채택 (Commander 결정)

---

## 1. 물리 계층 (배선)

M12 x1, 5-pin 커넥터 (데이터시트 확정):

| 핀 | 색 | 기능 |
|---|---|---|
| 1 | RD(빨강) | +U_B 전원 + (**10~30V DC**, 역극성 보호) |
| 3 | BK(검정) | −U_B / GND (전원 −) |
| 2 | YE(노랑) | **A (+RS485)** |
| 4 | GR(초록) | **B (−RS485)** |

- RS485 신호선만으로는 구동 불가 → **별도 10~30V DC 전원 필수**.
- 버스 양끝 120Ω 종단저항 권장.

## 2. 링크 계층 (시리얼 파라미터)

| 항목 | 값 | 확정 방법 |
|---|---|---|
| 보드레이트 | **9600** | 프로빙(9600 우선 시도 → 응답) |
| 데이터/패리티/스톱 | **8 / N / 1** | 표준값으로 통신 성립 |
| 프로토콜 | Modbus-RTU | 데이터시트 명시 |
| 슬레이브 주소 | **1** | 프로빙 |

## 3. 레지스터 맵 (역설계)

### 3.1 Input Registers — Function Code `0x04` (실시간 데이터)

유효 주소 범위: `0x0000 ~ 0x0003` (그 이상은 Modbus 예외 응답).

| 주소 | 이름 | 단위/의미 | 근거 |
|---|---|---|---|
| `0x0000` | 거리(미세) | 1 count = **0.1mm**, 대상 없으면 0 | reg0 = reg2 × 1.709, 실측 대조 |
| `0x0001` | 상태값(미상) | 관측 17~22, **대상 유무와 무관하게 유지** | 대상 제거 시에도 안 떨어짐 → 신호강도 아님 |
| `0x0002` | **거리(정식)** | 1 count = **0.17mm**, 대상 없으면 0 | 데이터시트 분해능 0.17mm와 일치 |
| `0x0003` | 미사용(추정) | 항상 0 | 관측 |

**경계/무효 거동 (실측 확정, 2026-07-09):**
- **`reg2 = 0` = 유효 측정 없음** — 대상 없음 / 빔 이탈 / 최대(>1000mm) 초과 모두 0.
  거리 0mm는 물리적으로 불가(사각 60mm)하므로 0을 무효 센티넬로 사용.
- **사각지대 근접(<~70mm)**: 실제 거리를 반영 못 하고 최소값 부근(reg2 ~400~480)에
  **클램핑**되며 지터가 사라짐(값이 얼어붙음). 더 가까워져도 값이 작아지지 않음.
- SDK 는 `reg2==0 → NoTarget`, `[min,max] 밖 → OutOfRange` 로 걸러낸다.

**권장**: `0x0002`를 정식 거리 출력으로 사용.
`거리(mm) = reg0x0002 × 0.17` (오프셋 0, 데이터시트 기준).

실측 예:
| reg 0x0002 | × 0.17mm | 실제 거리 |
|---|---|---|
| 602 | 102.3mm | ≈ 10.2cm |
| 621 | 105.6mm | ≈ 10.6cm |
| 2096 | 356mm | ≈ 35.6cm |

### 3.2 Holding Registers — Function Code `0x03` (설정, 부분 파악)

`0x0000~0x000F` 읽기 가능. 대부분 0이며 `0x0002 = 3` 관측.
설정용 레지스터로 추정되나 **의미 미확정**(거리 데이터 아님).
슬레이브 주소/보드레이트 변경 레지스터 위치는 **미검증**(향후 확인 대상).

## 4. 프레임 예시

거리 읽기 요청 (슬레이브 1, func 0x04, reg 0x0002, 1개):
```
요청 : 01 04 00 02 00 01 90 0A      (마지막 2바이트 = CRC16, LSB first)
응답 : 01 04 02 [hi] [lo] [crc_lo] [crc_hi]
       예) 01 04 02 02 5A 38 6B  → 0x025A = 602 → 602×0.17 = 102.3mm
```

- CRC16: 다항식 반사형 0xA001, 초기값 0xFFFF, 프레임에 **하위바이트 먼저** 부착.
- 16비트 레지스터는 빅엔디안 `[hi][lo]`.

## 5. 미확정/향후 검증 항목

- 오프셋: 현재 0 가정(데이터시트 기준). 정밀 지그 2점 실측 시 재확인 가능.
- ~~범위 밖 거동~~ → **확정됨(3장 참조): reg2=0 무효, 사각 근접 클램핑.**
- ~~`0x0001` 신호강도 여부~~ → **신호강도 아님 확정**(대상 제거 시에도 유지). 정확한 의미는 미상.
- Holding 레지스터 의미 및 주소/보드레이트 변경 방법 미검증.

## 6. 재현 방법 (프로빙 도구)

```bash
./build/lgub_probe scan  --port /dev/ttyUSB0            # 주소·보드레이트
./build/lgub_probe regs  --port /dev/ttyUSB0 --baud 9600 --id 1 --func 4
./build/lgub_probe watch --port /dev/ttyUSB0 --baud 9600 --id 1 --reg 0 --func 4 --count 4
```
