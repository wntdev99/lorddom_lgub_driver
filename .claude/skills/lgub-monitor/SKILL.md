# LGUB 거리 실시간 터미널 모니터

`lgub monitor` 는 한 줄이 제자리에서 계속 갱신되는 라이브 화면이다:
```
  30.5 cm  [#######.................] 유효 | 유효 100% | 5/s
```
(현재 거리 cm / 0~1m 막대 게이지 / 상태 / 누적 유효율 / 초당 갱신수)

## 핵심: 라이브 화면은 "사용자 터미널"에서 돌아야 한다
제자리 갱신(`\r`) 화면은 Claude가 백그라운드 도구로 실행하면 사용자에게 실시간으로
보이지 않는다(출력만 캡처됨). 따라서 **사용자가 자기 터미널에서 직접 실행**하도록 안내한다.

Claude Code 프롬프트에 아래처럼 `!` 를 붙여 입력하면 현재 세션 터미널에서 실행된다:
```
!cd <저장소경로> && ./build/lgub monitor
```
- 종료: **Ctrl+C**
- 갱신 속도: `--hz 20` (기본 10Hz)
- 포트 강제: `--port /dev/ttyUSB0` (기본 자동탐지)

## 안내 순서
1. 바이너리 확인. 없으면 빌드:
   `cmake -S . -B build >/dev/null && cmake --build build -j >/dev/null`
2. 사용자에게 위 `!./build/lgub monitor` 실행을 안내한다(원하면 `--hz`/`--port` 옵션 설명).
3. "Ctrl+C 로 멈춥니다"를 함께 알린다.

## 화면 읽는 법 (사용자에게 설명)
- `유효` + cm 값 → 현재 그 거리에 물체가 있음
- `대상없음` → 유효 범위(약 6cm~1m)에 잡히는 물체 없음
- `범위밖` → 측정값이 범위를 벗어남(무시)
- `통신오류` → 연결 문제. 케이블을 다시 꽂으면 자동 재접속 시도. 안 되면 `lgub doctor`

## 참고
- 값을 파일로 남기며 보고 싶으면 `lgub log`, 프로그램이 값을 소비하려면 `lgub stream --json`.
- 단발 확인은 `lgub measure`. (이들은 lgub-sensor 스킬이 담당)
