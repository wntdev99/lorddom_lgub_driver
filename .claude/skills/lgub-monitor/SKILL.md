# LGUB 거리 실시간 보여주기

**핵심 원칙: 사용자는 터미널을 만지지 않는다.** 하드웨어팀·비전문가는 터미널·명령어를
모른다. 그러니 `!` 명령을 치라고 시키지 말고, **Claude가 대신 센서를 읽어 대화창에
값을 계속 보고**한다.

## 준비
`build/lgub` 확인, 없으면 빌드:
`cmake -S . -B build >/dev/null && cmake --build build -j >/dev/null`

## 방법 A — 기본 (터미널 불필요, Claude가 보고)

**한 번/몇 번 확인**: `./build/lgub measure --json` 실행 → JSON 파싱 →
"현재 약 30cm(0.30m) 앞에 물체가 있습니다" 처럼 사람 말로 보고. m·cm 병기.

**계속 지켜보기(연속 보고)**: `/loop` 로 주기 실행해 매번 현재 거리를 보고한다.
- 예: 사용자가 "거리 계속 보여줘" → `/loop 3s` 로 3초마다 `lgub measure --json` 실행,
  매 회 현재값을 채팅에 갱신 보고("현재 30.5cm", "현재 12.1cm" ...).
- 값이 유효하지 않으면 "대상 없음 / 범위 밖 / 통신오류(연결 확인)"로 알려준다.
- 사용자가 "그만/멈춰" 하면 루프를 종료한다.
- 주기를 사용자가 원하는 대로 조정(1~5초 권장; 너무 짧으면 대화가 시끄러움).

**고빈도로 잠깐 관찰(예: 20초간 촘촘히)**: 백그라운드 스트림을 파일로 받고 요약한다.
```bash
./build/lgub stream --json --hz 5 --sec 20 > /tmp/lgub_live.jsonl 2>&1   # run_in_background
```
끝나면 파일을 읽어 "20초간 평균 X cm, 최소~최대" 로 요약(또는 마지막 몇 개 스냅샷).

## 방법 B — 선택 (사용자가 스스로 라이브 TUI를 원할 때만)
제자리 갱신되는 화면을 자기 터미널에서 직접 보고 싶다고 **명시**하면 그때만 안내한다:
`!cd <저장소> && ./build/lgub monitor` (Ctrl+C 종료). 그 외에는 방법 A 를 쓴다.

## 값 해석 (사람 말로)
- 유효 + 거리 → "현재 약 N cm (= M m)"
- NoTarget → "유효 범위(약 6cm~1m)에 잡히는 물체가 없습니다"
- OutOfRange → "측정값이 범위를 벗어나 무시"
- 통신오류(Timeout 등) → "연결 문제. lgub-connection 으로 점검하세요"

## 참고
- 파일로 남기려면 `lgub log`(→ lgub-sensor), 단발 확인은 `lgub measure`.
- 연결 자체가 의심되면 lgub-connection 스킬로 먼저 점검.
